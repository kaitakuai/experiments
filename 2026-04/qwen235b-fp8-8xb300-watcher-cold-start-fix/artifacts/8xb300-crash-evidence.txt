6. Resolution path
================================================================================

  k4 keeps the same per-card vLLM perf as k3 and adds two layered
  defenses (env-driven, defaults preserve upstream behavior):

    runner.WAIT_FOR_SERVER_TIMEOUT  →  60 min via VLLM_RUNNER_TIMEOUT=3600
    watcher.watch_managers          →  session-aware first-healthy grace
                                       via WATCHER_GRACE_FIRST_HEALTHY=1

  Plus tensor-parallel-size=1 forced (override-resistant) so the
  network-node-supplied TP=2 is overwritten with the throughput-optimal
  TP=1 layout (8× independent instances on 8×B300).
